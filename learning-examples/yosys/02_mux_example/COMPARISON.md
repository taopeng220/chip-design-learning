# MUX 综合对比实验

## 原始 RTL
```verilog
assign y = sel ? b : a;
```

---

## 方式 1：保留 MUX 结构

### 综合命令
```bash
yosys synth_with_mux.ys
```

### 综合结果
```verilog
$_MUX_ _0_ (
  .A(a),    // sel=0 → 选 a
  .B(b),    // sel=1 → 选 b
  .S(sel),
  .Y(y)
);
```

**统计：**
- 1 个 MUX 门
- 面积小，结构清晰

---

## 方式 2：分解成基本门

### 综合命令
```bash
yosys synth_no_mux.ys
```

### 综合结果（逻辑：y = (sel & b) | (!sel & a)）
```verilog
$_NOT_ gate1 (.A(sel), .Y(n1));        // n1 = !sel
$_AND_ gate2 (.A(n1), .B(a), .Y(n2)); // n2 = !sel & a
$_AND_ gate3 (.A(sel), .B(b), .Y(n3));// n3 = sel & b
$_OR_  gate4 (.A(n2), .B(n3), .Y(y)); // y = n2 | n3
```

**统计：**
- 1 个 NOT 门
- 2 个 AND 门
- 1 个 OR 门
- 总共 4 个门

---

## 🤔 为什么会有两种方式？

### 在 FusionCompiler 中的对应

**类比：**
```
FusionCompiler:
- 工艺库有 MUX2 单元 → 直接用 MUX2
- 工艺库没有 MUX → 用 NAND/NOR 组合

Yosys:
- 目标有 MUX 原语 → 保留 MUX 结构
- 目标只有基本门 → 分解成 AND/OR/NOT
```

---

## 🎯 实际应用场景

### 场景 1：FPGA 综合
- FPGA 有专用 MUX 资源（LUT 可以高效实现 MUX）
- → **保留 MUX 结构更好**

### 场景 2：ASIC 综合（标准单元库）
- 大多数工艺库有 MUX2、MUX4 标准单元
- → **保留 MUX 结构更好**

### 场景 3：纯逻辑优化
- 如果要进一步优化（布尔代数）
- → **分解成基本门更灵活**

---

## 📊 对比总结

| 指标 | 保留 MUX | 分解成基本门 |
|------|---------|-------------|
| **门数** | 1 | 4 |
| **面积** | 小 | 大 |
| **延迟** | 低（单级） | 高（多级） |
| **适用** | FPGA/ASIC | 纯逻辑学习 |

## 💡 关键启示

### 在 FusionCompiler 中：
- 您不需要担心这些，工具会**自动选择**最优实现
- 工具知道目标库有什么单元

### 在 Yosys 中：
- 您可以**控制**综合策略
- 通过不同命令选项，看到不同实现方式
- **这正是开源工具的学习价值！** ✨
